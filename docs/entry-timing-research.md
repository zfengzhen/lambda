# 入场限价优化研究报告

**日期**：2026-04-01
**标的**：TQQQ（3x 杠杆 QQQ）
**关联策略**：Lambda 策略（A/B 分层 Sell Put）
**数据来源**：本地 DuckDB，期权日线 2024-04 ~ 2026-03

---

## 研究目标

Lambda 策略在周一收盘后确定行权价和到期日。本研究回答：

> 周二开盘时，应该用什么价格挂卖 Put 限价单，才能最大化总权利金？

---

## 数据概况

| 项目 | 数值 |
|------|------|
| 信号周总数 | 46 笔 |
| 有完整期权日线的周 | 29 笔 |
| 缺失原因 | 深虚值 Put 在信号周流动性不足，无成交记录 |

缺失的 17 笔均为 OTM 10-15% 的 Put，到期前 3 周在 Flat Files 中无任何成交。这不是数据同步问题，而是合约本身流动性限制。

---

## 核心结论

### 1. 最优限价倍数 k = 1.00（即市价单）

对 k ∈ [0.5, 3.0] 全量扫描，总权利金在 k=1.00 时最大。提高限价（k > 1）导致的不成交损失 > 单笔溢价收益。

```
全局 k_star = 1.00
成交率:     75.9%（22/29 笔）
总权利金:   34.47
```

### 2. 分层分析

| 层级 | 笔数 | 最优 k | 成交率 | 总权利金 |
|------|------|--------|--------|----------|
| A    | 18   | 1.00   | 72%    | 23.96    |
| B2   | 8    | 1.70   | 62%    | 8.62     |
| B3   | 3    | 1.05   | 100%   | 5.09     |

B2 层单独看 k=1.70 更优，但仅 8 笔样本，不足以作为分层参数依据。

## 实盘操作建议

**推荐方案：周一盘后挂限价单 = 周一期权收盘价 P**

```
周一收盘 → python run.py → 获取层级 / 行权价 / 到期日
周一盘后 → 查对应 Put 收盘价 P（富途/IBKR）
         → 挂限价卖单 = P，GTC 有效至周五收盘
周五收盘 → 未成交则自动取消，等下周新信号
```

**为什么不用市价单？**
- 68% 的周二开盘权利金比周一收盘低（平均 -3.9%）
- 损耗来自 theta decay（~1.8%）+ 流动性/IV 损耗（~2%）
- 限价单以 78.6% 的概率在周内成交，避免了开盘时的即时损耗

**为什么不挂更高的限价？**
- k > 1.0 的总权利金反而下降
- 不成交的损失 > 单笔多收的溢价

**未成交怎么办？**
- 21.4% 的周不会成交，放弃即可
- 未成交的周通常是标的持续上涨、Put 权利金一路走低
- 相当于自动过滤了"标的突然大涨"的不利周

---

## 局限性

1. **有效样本仅 29 笔**（17 笔因流动性缺失被排除），结论的统计显著性有限
2. **回测假设 week_high ≥ limit 即成交**，忽略实际买盘深度
3. **Theta decay 和 IV 变化未分离量化**，3.9% 的损耗构成为估算值
4. **未考虑保证金成本和交易手续费**
